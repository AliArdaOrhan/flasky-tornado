class flasky.app.FlaskyApp(ioloop=None, **settings)[source]¶

The Bant object provides a convenient API to configure Application It acts as a container for all plugins, crosscutting concern functions, objects and settings. On invocation of run method, Bant instance registers DynamicHandler classes for every endpoint that is defined by user.

Bant instance might be created at the __init__.py of your package. Another approach might be creating initialization closures to init API functions. For more information please refer to :ref:

from bant import Bant
app = Bant(**settings)
Parameters:
  • ioloop – IOLoop that will be used by application. By default Bant application uses asyncio.BaseEventLoop
  • settings – Application specific settings.

after_request(f)[source]¶

Registers a function to run after each request handler.

Function should take 2 argument an instance of handler.DynamicHandler and :type:`dict` of endpoint definition:

@app.after_request
async def add_cors_headers
    handler.set_header("Allow-Origin", "*")

It’s important to know that, if any error raised during pipeline, after request functions WILL NOT BE EXECUTED.

To execute a function after pipeline in every circumstances, use teardown_request()

after_request_funcs = None¶

A list of functions which will be called after the request handled by handler function. Function should take 2 parameters, a RequestHandler instance and endpoint definition.

Warning: These functions will not be called if any error occurs during the execution of handler. To run piece of code after request in every circumstances please please check teardown_request()

To register a function use the after_request() decorator

api(host='.*$', endpoint=None, method=None, **kwargs)[source]¶

A decorator which will be used to register a handler for given endpoint. Parameters are default tornado.web.RequestHandler parameters. Regexes can be used at endpoint definition and will be injected to as second parameter:

@app.api(
    endpoint="/api/user",
    method="POST"
)
async def create_user(handler, *args, **kwargs):
    handler.write("hello world")

@app.api(
    endpoint="api/user",
    method="GET"
)
async def get_users(handler, *args, **kwargs):
    handler.write({"user": "Asimov"})
Parameters:
  • host – Host Address in which this handler function will be registered.
  • endpoint – Endpoint address for which this handler function will be executed.
  • method – HTTP Method(POST, GET, PUT etc..) for which this handler function will be executed
:param options:” Options that will be attached to handler function

class flasky.handler.DynamicHandler(application, request, **kwargs)[source]¶

DynamicHandler is the main class where magic happens.

In simplest word, handler provides convenient methods to access request object, methods to prepare response and executes all the pipeline from top to bottom.

DynamicHandler class delegates all HTTP calls to _handle() method to execute programatic pipeline.

Any plugin may have access to dynamic handler instance of current request, and may manipulate it to provide new functionalities.

Plugin developers may use before_request() function to provide an entry point for plugin that they develop.
